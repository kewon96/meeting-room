# Docker Swarm vs Kubernetes

> 한 줄 요약: 컨테이너 오케스트레이션 도구인 Docker Swarm과 Kubernetes(k8s)의 개념·아키텍처·차이·선택 기준을 정리한다.

## 배경
컨테이너를 단일 호스트에서 `docker run` 으로 굴리는 단계를 넘어가면 **여러 노드에 걸쳐 컨테이너를 배치·재시작·확장·네트워킹·롤링업데이트** 하는 일을 누군가 대신 해줘야 한다. 그 역할을 하는 게 컨테이너 오케스트레이터다. 시장에는 크게 두 진영이 있다 — Docker가 자체 제공하는 **Swarm**, 그리고 사실상의 표준이 된 **Kubernetes**. 이 주제는 두 도구가 각각 무엇을 어떻게 해결하는지, 어떤 경우 어느 쪽을 고르는지 설명하기 위해 열렸다.
